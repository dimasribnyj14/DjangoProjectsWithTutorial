Тема JavaScript'a: LocalStorage

Скопійовано в буфер обміну

Дискусія

Window.localStorage

Властивість localStorage (переклад з англ. "локальне сховище") дозволяє отримати доступ до об'єкта Storage. localStorage аналогічно властивості sessionStorage. Різниця лише в тому, що властивість sessionStorage зберігає дані протягом сеансу (до закриття браузера), на відміну від даних, що знаходяться у властивості localStorage, які не мають обмежень за часом зберігання та можуть бути видалені лише за допомогою JavaScript. Слід зазначити, що дані, збережені як у localStorage, і у sessionStorage, є специфічними для протоколу сторінки. Ключі та значення завжди рядки (так само, як і об'єкти, цілі ключі автоматично будуть перетворені на рядки).

Синтаксис

myStorage = window.localStorage;

Значення

Об'єкт Storage, який використовується для доступу до локального сховища.

Винятки

SecurityError

Запит до сховища порушує дозвіл політик, або джерело зберігання не є коректною комбінацією схема/хост/порт (таке може статися, якщо джерелом для зберігання є file: або data: схеми, наприклад). Як ще один приклад появи помилки користувач через конфігурацію браузера заборонив зберігання даних для деяких джерел.

Наприклад

Наступний код показує приклад доступу до локального об'єкта Storage для поточного домену і додає дані до нього за допомогою Storage.setItem().

localStorage.setItem('myLesson', 'Math');

Зчитувати дані з localStorage для певного ключа можна так:

let lesson = localStorage.getItem('myLesson');

Видаляти дані можна так (після видалення поверне undefined):

localStorage.removeItem('myLesson');

Для видалення всіх записів, тобто повного очищення localStorage, використовуйте:

localStorage.clear();
